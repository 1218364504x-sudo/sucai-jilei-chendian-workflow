# Codex 接手提示词

把下面这段复制到新的 Codex 对话中，替换其中的路径和链接。

```text
请基于我现有的故事素材流程继续操作，不要从零设计。

当前目标：
Codex 负责挖掘、去包装、核验、筛选、整理故事；
Obsidian 作为故事卡片的长期素材库；
飞书多维表格用于快速评级和二次筛选；
飞书知识库用于沉淀 4-6 星精品故事。

Obsidian 素材库：
<你的 Obsidian vault 路径>

故事卡目录：
<你的故事卡目录>

飞书多维表格：
<你的飞书多维表格链接>

飞书精选故事知识库：
<你的飞书知识库链接>

同步规则：
1. 同步唯一标识是「文件路径」，不要改成别的主键。
2. Obsidian 修改故事卡后，同步到飞书表格。
3. 飞书修改「星级评级」后，写回 Obsidian 的「星级」字段。
4. 飞书表格只用于评级，日常只看标题、故事简介、星级评级。
5. 4-6 星进入精选知识库。

星级规则：
1-2 星：暂不满意，进入删除候选。
3 星：待选，进入复审。
4-5 星：优秀，进入精选。
6 星：特别喜欢，进入精选。

当前收集偏好：
1. 优先服务微电影、短片和小切口人物故事。
2. 每批正式故事默认约 70% 偏短片/微电影，约 20% 中等体量，0-5% 宏大历史或大场景。
3. 地区配额不能压过质量门槛；新闻不够强时，可以找已核验的历史档案、人物报道、口述史、纪实文本或长报道。
4. 标题和简介一样要走完整流程：外文/非中国素材走 `stop-slop -> humanizer -> DeepSeek Pro -> shuorenhua`；中国素材走 `DeepSeek Pro -> shuorenhua`。标题从动作、声音、道具、关系代价或结尾余味里提取，不写成功能说明或新闻标题，并留下标题审计记录。
5. 不再主动收集战争、军事、特工、情报网络、战争英雄、种族制度冲突、族裔对抗、种族隔离和民权斗争类素材。

故事简介要求：
简介必须写成完整可读的故事，说明人物、时间、地点、发生了什么、后来怎样、具体结果是什么。
不要写成抽象概念或价值判断。
国外/非中国素材简介必须走固定流水线：`stop-slop -> humanizer -> DeepSeek Pro -> shuorenhua`。先用 `stop-slop` 和 `humanizer` 按英文去 AI 味规则清理来源和事实资料，再由 DeepSeek Pro 重写中文故事简介，最后用 `shuorenhua` 做中文验收。中国素材默认走 `DeepSeek Pro -> shuorenhua`。没有可查询的 DeepSeek 审计记录，不要声称故事已经由 DeepSeek 重写。
审计记录至少要能查到模型、Key 指纹、流水线版本、流水线步骤、来源哈希、事实资料哈希、输出哈希、末句和质量结果。
用审计记录确认 DeepSeek 是否真实参与；余额变化可能延迟，只有需要核对消耗或账号时再查。翻译腔、评论式末句和词表命中只作为质量提醒，不作为机械拦截。

收集筛选要求：
1. Codex 不是新闻搬运器，而是故事筛片人。
2. 每个候选先去包装：剔除标题党、宣传话术、营销口号、媒体情绪和夸张形容词。
3. 每个候选先拆事实：区分已发生事实、当事人说法、机构说法、媒体判断和待核验信息。
4. 宣传报道、通稿、表彰稿、品牌故事、热点新闻默认降权；只有能挖出具体过程、矛盾、失败、代价和可拍细节时，才考虑入库。
5. 一句话新闻钩子不是可拍性；展开后只有等待、通报、检查和解释的，跳过。
6. 正式故事必须有明确改编拍摄点：时间链条、人物/群体/地点/物件、动作、空间、道具、声音、关系或处境变化。
7. 正式故事入库前过四道门：人物门、关系门、时间门、影像门。缺失明显的，只放收件箱、场景灵感或跳过。

请先读取流程交接说明和自检报告，再继续操作。
```
