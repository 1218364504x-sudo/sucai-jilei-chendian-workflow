# 飞书多维表格设计

## 定位

飞书多维表格只做一件事：快速评级。

不要把它设计成完整资料库。完整资料保存在 Obsidian，精品故事沉淀到飞书知识库。

## 日常视图核心字段

日常评级视图只需要 4 列：

| 字段 | 用途 |
|---|---|
| 标题 | 快速判断故事主题 |
| 故事简介 | 读完整故事梗概 |
| 拍摄可行性 | 判断当前阶段好不好拍 |
| 星级评级 | 用户最终打星 |

## 后台字段

后台可以保留同步所需字段，但不放在日常评级视图里：

- 文件路径
- 分类
- 国家地区
- 来源类型
- 来源链接
- 适合形式
- 开发优先级
- 低成本切口
- AI建议星级
- AI建议理由
- 知识库收纳状态
- 最后修改时间

## 推荐视图

| 视图 | 筛选规则 | 用途 |
|---|---|---|
| 01_待评级 | 星级评级为空 | 日常主入口 |
| 02_3星复审 | 星级评级 = 3 星 | 每周复审 |
| 03_4-5星精选 | 星级评级 = 4 或 5 星 | 优秀故事 |
| 04_6星特别喜欢 | 星级评级 = 6 星 | 最高优先级 |
| 05_删除候选_1-2星 | 星级评级 = 1 或 2 星 | 每周删除复核 |
| 99_后台全量_同步排查 | 全量 | 排查同步问题 |

## 自动归类逻辑

用户在飞书修改 `星级评级` 后：

1. 当前记录从 `01_待评级` 消失。
2. 进入对应星级视图。
3. 星级同步回 Obsidian 的 `星级` 字段。
4. 如果是 4-6 星，进入精选区和知识库。
5. 自检报告更新统计。

## 最容易出错的点

### 1. 用错主键

必须使用 `文件路径` 作为同步唯一标识。不要用标题。

### 2. 待评级数量不对

待评级不是 `AI建议星级` 为空，而是正式 `星级评级` 为空。

### 3. 日常视图字段太多

字段越多，评级越慢。飞书表格不是资料库，只保留判断所需信息。

保留 `拍摄可行性` 是为了避免把高价值但难拍的故事误打低分。星级只评故事价值，拍摄难度单独看。

### 4. 同时在 Obsidian 和飞书改同一字段

需要明确冲突策略。推荐默认 Obsidian 优先，飞书只负责 `星级评级`。
