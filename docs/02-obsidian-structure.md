# Obsidian 结构

## 目录原则

故事卡只按一级分类存放，不按星级、不按日期、不按是否精选重复复制。

星级变化由字段、视图和索引自动呈现，文件本体不移动。

## 首页

`index.md` 只放最重要入口：

- 精选故事
- 故事工作台
- 自检报告
- 最近统计

首页不要变成大杂烩。

## 故事工作台

`00_故事工作台.md` 是日常入口，显示：

- 待评级
- 3 星复审
- 4-6 星精选
- 1-2 星删除候选

工作台表格不要只显示星级和简介，还要显示 `拍摄可行性` 和 `开发优先级`。这样高价值但难拍的故事会被看成“高价值储备”，而不是被误放进低星或删除候选。

## 精选区

`00_0_精选故事/` 用来让用户优先看到好故事。

推荐包含：

- `精选故事.md`：按日期或星级聚合 4-6 星故事。
- `精选故事浏览.base`：Obsidian Bases 视图。
- `删除候选.md`：每周清理时使用。
- `周优化.md`：每周复审清单。
- `月优化.md`：月度 Top 10 和结构整理。

精选区的 Base 视图建议显示：

- 故事
- 星级
- 分类
- 拍摄可行性
- 开发优先级
- 低成本切口
- 故事简介
- 来源可信度

删除候选和 3 星复审也要显示 `拍摄可行性` 和 `开发优先级`，避免把“当前难拍”和“故事不好”混在一起。

## 故事卡命名

```text
YYYY-MM-DD_分类_短标题.md
```

示例：

```text
2026-05-31_社会现实类_南昌医院旁的一元抗癌厨房.md
```

标题要让人一眼看懂故事核心，不要只放外文原名或抽象概念。

## 标题规则

标题要像一个可拍瞬间，从故事里的动作、声音、道具、关系代价或结尾余味里提取。

标题和简介一样需要走完整去 AI 味流水线：

- 外文/非中国素材：`stop-slop -> humanizer -> DeepSeek Pro -> shuorenhua`
- 中国素材：`DeepSeek Pro -> shuorenhua`

标题也要留下可追溯审计记录。没有标题审计记录，不算完成标题去 AI 味流程。

推荐：

- 人物或群体 + 动作 + 关键物件/对象。
- 长期动作 + 情感对象。
- 空间/道具 + 命运变化。

避免：

- 功能说明式标题。
- 新闻标题式标题。
- 只写事件标签、地点标签或主题标签。
- 震撼、泪目、奇迹、传奇等包装词。
