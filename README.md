# 素材积累沉淀工作流

一套面向导演、编剧、纪录片创作者和内容策划的故事素材收集流程。

当前版本：**v2**。这一版把流程从“故事收集与评级”升级为“电影化筛选、短片小切口优先、简介重写审计和定期自检”的完整工作流。

它把 **Codex / AI 助手、Obsidian、飞书多维表格、飞书知识库** 分成四个不同角色：

- **AI 助手**：负责挖掘、去包装、核验、筛选、改写、补全故事卡和维护流程。
- **Obsidian**：作为长期素材库，保存完整故事卡、来源、伦理提醒和创作观察。
- **飞书多维表格**：只做快速评级，不承担资料库功能。
- **飞书知识库**：沉淀已完成星级和拍摄可行性判断的 4-6 星精品故事，方便后续阅读和开发。

这个流程的核心目标不是“多收集”，而是让素材持续变少、变清楚、变可用。它特别强调剔除标题党、宣传通稿、眼球新闻和一句话噱头，只保留真正有电影化、微电影化潜力的故事。

## v2 更新重点

- 收集标准改为电影改编价值优先，不再追求热点和分类覆盖。
- 每条正式故事入库前先过人物门、关系门、时间门、影像门。
- 当前偏向微电影、短片和小切口人物故事。
- 标题和简介一样走去包装、DeepSeek Pro 和 `shuorenhua` 流程，再从动作、声音、道具、关系代价或结尾余味里提取。
- 星级只评故事价值，拍摄难度单独进入 `拍摄可行性` 和 `开发优先级`。
- Obsidian 工作台、精选 Base、周/月优化、删除候选、飞书其它列表和飞书知识库都要同步显示拍摄可行性判断；两项未填完时不做最终归类。
- 外文/非中国素材简介明确走 `stop-slop -> humanizer -> DeepSeek Pro -> shuorenhua`。
- 中国素材默认走 `DeepSeek Pro -> shuorenhua`。
- DeepSeek 重写必须留下审计记录；没有审计记录，不算完成重写。
- 飞书多维表格继续只做快速评级，不作为完整资料库。

## 适合谁

- 导演、编剧、纪录片创作者
- 短视频、栏目、播客、长内容策划
- 需要长期积累真实事件、人物故事、社会议题素材的人
- 想把 AI 搜集结果沉淀成稳定资料库的人

## 设计原则

1. **AI 不是新闻搬运器，而是故事筛片人。**
2. **先去包装，再拆事实，再判断可拍性。**
3. **Obsidian 是资料库，飞书表格只是评级器。**
4. **故事卡只存一份，不按星级复制或移动。**
5. **用字段和视图归类，不用文件夹乱搬素材。**
6. **标题和简介都必须走去包装、DeepSeek Pro 和中文验收流程。**
7. **星级和拍摄可行性都填完后，4-6 星才进入精选知识库，1-2 星才进入删除候选。**
8. **国外/非中国素材必须走 `stop-slop -> humanizer -> DeepSeek Pro -> shuorenhua`，并留下可查询审计记录。**
9. **任何自动同步都必须有自检报告。**
10. **当前收集优先服务微电影、短片和小切口人物故事，不追求新闻覆盖面。**

## 星级规则

| 星级 | 含义 | 去向 |
|---|---|---|
| 1-2 星 | 暂不满意 | 两项完整后进入删除候选 |
| 3 星 | 待选 | 两项完整后进入复审或低成本练手 |
| 4-5 星 | 优秀 | 两项完整后进入 Obsidian 精选 + 知识库 |
| 6 星 | 特别喜欢 | 两项完整后进入 Obsidian 精选 + 知识库 + 6 星视图 |

星级只评故事价值，不评拍摄难度。好故事即使成本高、调度难，也可以给高星；拍摄难度单独写入 `拍摄可行性`。

## 拍摄可行性

用 `拍摄可行性` 和 `开发优先级` 把“故事好不好”和“现在好不好拍”拆开。

| 拍摄可行性 | 含义 |
|---|---|
| A_低成本可拍 | 一个房间、几个人、一件物、一个小空间或少量日常场景就能成立。 |
| B_中等成本 | 需要少量场景、演员、调度或基础制作资源。 |
| C_高难度 | 需要年代还原、跨地域、大量群众、特殊场景、复杂版权或较高制作成本。 |
| D_暂不适合直接拍 | 更适合长期调研、纪录片开发、主题储备或灵感使用。 |

| 组合 | 开发优先级 |
|---|---|
| 未打星或未填拍摄可行性 | 待完整判断 |
| 高星 + A/B | 优先开发 |
| 高星 + C/D | 高价值储备 |
| 中星 + A | 低成本练手 |
| 中星 + B/C/D | 暂存 |
| 低星 + C/D | 删除候选或暂存 |

每张故事卡保留 `低成本切口`。它用来保护“好但难拍”的故事：不直接拍全貌，而是提炼一个可执行入口，例如一通电话、一件遗物、一个房间、一次等待、一段采访、一个孩子的视角或事后某个人回到现场。

归类门槛改为两项同时完成：`星级评级` 判断故事价值，`拍摄可行性` 判断当前拍摄难度。任意一项为空，`开发优先级` 都是 `待完整判断`，故事留在飞书 `01_待完整判断`，不进入复审、精选、知识库或删除候选。两项都填完后，系统才自动生成最终开发优先级。

这些字段不能只存在于故事卡里。它们还应该出现在：

- Obsidian 工作台的待完整判断、3 星复审、4-6 星精选和删除候选表格。
- Obsidian 精选 Base 和评分 Base。
- 周优化、月优化和删除候选页面。
- 飞书多维表格的待完整判断、优先开发、高价值储备、低成本练手、暂存和删除候选等辅助视图。
- 飞书知识库的日期页正文。

日常飞书评级视图仍保持轻量，只看 `标题`、`故事简介`、`拍摄可行性`、`星级评级` 四列；其它视图再显示 `开发优先级` 和必要的分类、理由字段。

## 简介重写流水线

`故事简介` 是飞书评级和精选知识库里最常被阅读的字段，不能只做翻译、摘要或价值判断。简介要写成完整故事：谁、什么时候、在哪里、原本怎样、发生了什么转折、后来做了什么、留下什么具体结果。

### 外文/非中国素材

外文和非中国素材统一走：

```text
stop-slop -> humanizer -> DeepSeek Pro -> shuorenhua
```

具体顺序：

1. `stop-slop`：先清理英文源材料和事实资料里的 AI 写作套路，去掉填充句、二元对比、被动堆砌、空泛意义、远距离旁白和金句化结尾。
2. `humanizer`：继续检查宣传腔、显著性夸大、无源权威、not X but Y、规则三连、-ing 浮夸分析、广告式措辞和模板感。
3. `DeepSeek Pro`：根据清理后的事实资料，用中文按事实链重写故事简介，不逐句翻译，不保留外文语序。
4. `shuorenhua`：做中文验收，清掉翻译腔、欧化长句、抽象主语、旁白评论、价值拔高和评论式末句。

### 中国素材

中国素材默认走：

```text
DeepSeek Pro -> shuorenhua
```

DeepSeek Pro 负责按事实链重写，`shuorenhua` 负责中文验收。中国素材不默认套 `stop-slop` 和 `humanizer` 的英文前置流程，除非来源材料本身是英文，或明显带有英文 AI 写作模板。

### 审计和验收

每次 DeepSeek 真实调用都要写入审计记录。有效记录至少要包含模型、Key 指纹、流水线版本、流水线步骤、来源哈希、事实资料哈希、输出哈希、末句和质量结果。没有有效审计记录，不能声称这条故事已经由 DeepSeek 重写。

默认看审计记录确认 DeepSeek 是否参与，不默认等待余额刷新。翻译腔、评论式末句和词表命中只是低级质量提醒，不机械阻断；真正需要阻断的是简介缺失、简介过短、审计记录缺失、哈希不匹配、模型或流水线错误、YAML 简介和正文第一段明显不一致。

## 标题重写流水线

标题和简介一样要走完整流程，只是输出更短。标题不能停留在新闻标题、资料标签、宣传口号、翻译腔或硬凑动词上。

外文/非中国素材标题走：

```text
stop-slop -> humanizer -> DeepSeek Pro -> shuorenhua
```

中国素材标题走：

```text
DeepSeek Pro -> shuorenhua
```

标题验收标准：

- 必须基于已核验事实或已审计通过的事实链简介。
- 最好能看见一个人/群体、一个动作或声音、一个道具或空间。
- 可以有一点诗意，但诗意必须来自真实动作、声音、道具、空间、关系代价或结尾余味。
- 不使用震撼、泪目、奇迹、传奇等包装词。
- 不写功能说明、主题标签、新闻标题或硬凑动词。
- 标题也要留下审计记录；没有标题审计记录，不算完成标题去 AI 味流程。

## 最小可用流程

1. AI 助手先去掉新闻/宣传包装，筛掉噱头和夸张话术。
2. AI 助手挖掘真正有可拍性的故事，写入 Obsidian 故事卡。
3. 按“标题重写流水线”和“简介重写流水线”处理标题和故事简介，并留下 DeepSeek 审计记录。
4. 故事卡同步到飞书多维表格。
5. 用户只在飞书里看 `标题`、`故事简介`、`拍摄可行性`、`星级评级` 四列。
6. 用户同时补 `星级评级` 和 `拍摄可行性`；星级写回 Obsidian。
7. 任意一项为空，故事留在 `01_待完整判断`，`开发优先级` 为 `待完整判断`。
8. 两项都填完后，系统再生成最终 `开发优先级`。
9. 3 星复审、4-6 星精选和 1-2 星删除候选只统计两项完整的故事，并同步显示 `拍摄可行性` 和 `开发优先级`。
10. 两项完整的 4-6 星进入精选视图和知识库，知识库正文保留 `拍摄可行性`、`开发优先级` 和 `低成本切口`。
11. 每次同步后生成自检报告。
12. 每周清理 1-2 星，复审 3 星。
13. 每月选 Top 10，整理主题、版权和伦理风险。
