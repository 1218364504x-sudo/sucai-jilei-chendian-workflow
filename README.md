# 素材积累沉淀工作流

一套面向导演、编剧、纪录片创作者和内容策划的故事素材收集流程。

当前版本：**v2**。这一版把流程从“故事收集与评级”升级为“电影化筛选、短片小切口优先、简介重写审计和定期自检”的完整工作流。

它把 **Codex / AI 助手、Obsidian、飞书多维表格、飞书知识库** 分成四个不同角色：

- **AI 助手**：负责挖掘、去包装、核验、筛选、改写、补全故事卡和维护流程。
- **Obsidian**：作为长期素材库，保存完整故事卡、来源、伦理提醒和创作观察。
- **飞书多维表格**：只做快速评级，不承担资料库功能。
- **飞书知识库**：沉淀 4-6 星精品故事，方便后续阅读和开发。

这个流程的核心目标不是“多收集”，而是让素材持续变少、变清楚、变可用。它特别强调剔除标题党、宣传通稿、眼球新闻和一句话噱头，只保留真正有电影化、微电影化潜力的故事。

## v2 更新重点

- 收集标准改为电影改编价值优先，不再追求热点和分类覆盖。
- 每条正式故事入库前先过人物门、关系门、时间门、影像门。
- 当前偏向微电影、短片和小切口人物故事。
- 标题从动作、声音、道具、关系代价或结尾余味里提取，避免新闻标题和资料标签。
- 外文/非中国素材简介明确走 `stop-slop -> humanizer -> DeepSeek Pro -> shuorenhua`。
- 中国素材默认走 `DeepSeek Pro -> shuorenhua`。
- DeepSeek 重写必须留下审计记录；没有审计记录，不算完成重写。
- 飞书多维表格继续只做快速评级，不作为完整资料库。

## 适合谁

- 导演、编剧、纪录片创作者
- 短视频、栏目、播客、长内容策划
- 需要长期积累真实事件、人物故事、社会议题素材的人
- 想把 AI 搜集结果沉淀成稳定资料库的人

## 设计原则

1. **AI 不是新闻搬运器，而是故事筛片人。**
2. **先去包装，再拆事实，再判断可拍性。**
3. **Obsidian 是资料库，飞书表格只是评级器。**
4. **故事卡只存一份，不按星级复制或移动。**
5. **用字段和视图归类，不用文件夹乱搬素材。**
6. **标题和简介必须像故事，不像资料摘要。**
7. **4-6 星才进入精选知识库，1-2 星进入删除候选。**
8. **国外/非中国素材必须走 `stop-slop -> humanizer -> DeepSeek Pro -> shuorenhua`，并留下可查询审计记录。**
9. **任何自动同步都必须有自检报告。**
10. **当前收集优先服务微电影、短片和小切口人物故事，不追求新闻覆盖面。**

## 星级规则

| 星级 | 含义 | 去向 |
|---|---|---|
| 1-2 星 | 暂不满意 | 删除候选 |
| 3 星 | 待选 | 复审视图 |
| 4-5 星 | 优秀 | Obsidian 精选 + 知识库 |
| 6 星 | 特别喜欢 | Obsidian 精选 + 知识库 + 6 星视图 |

## 简介重写流水线

`故事简介` 是飞书评级和精选知识库里最常被阅读的字段，不能只做翻译、摘要或价值判断。简介要写成完整故事：谁、什么时候、在哪里、原本怎样、发生了什么转折、后来做了什么、留下什么具体结果。

### 外文/非中国素材

外文和非中国素材统一走：

```text
stop-slop -> humanizer -> DeepSeek Pro -> shuorenhua
```

具体顺序：

1. `stop-slop`：先清理英文源材料和事实资料里的 AI 写作套路，去掉填充句、二元对比、被动堆砌、空泛意义、远距离旁白和金句化结尾。
2. `humanizer`：继续检查宣传腔、显著性夸大、无源权威、not X but Y、规则三连、-ing 浮夸分析、广告式措辞和模板感。
3. `DeepSeek Pro`：根据清理后的事实资料，用中文按事实链重写故事简介，不逐句翻译，不保留外文语序。
4. `shuorenhua`：做中文验收，清掉翻译腔、欧化长句、抽象主语、旁白评论、价值拔高和评论式末句。

### 中国素材

中国素材默认走：

```text
DeepSeek Pro -> shuorenhua
```

DeepSeek Pro 负责按事实链重写，`shuorenhua` 负责中文验收。中国素材不默认套 `stop-slop` 和 `humanizer` 的英文前置流程，除非来源材料本身是英文，或明显带有英文 AI 写作模板。

### 审计和验收

每次 DeepSeek 真实调用都要写入审计记录。有效记录至少要包含模型、Key 指纹、流水线版本、流水线步骤、来源哈希、事实资料哈希、输出哈希、末句和质量结果。没有有效审计记录，不能声称这条故事已经由 DeepSeek 重写。

默认看审计记录确认 DeepSeek 是否参与，不默认等待余额刷新。翻译腔、评论式末句和词表命中只是低级质量提醒，不机械阻断；真正需要阻断的是简介缺失、简介过短、审计记录缺失、哈希不匹配、模型或流水线错误、YAML 简介和正文第一段明显不一致。

## 最小可用流程

1. AI 助手先去掉新闻/宣传包装，筛掉噱头和夸张话术。
2. AI 助手挖掘真正有可拍性的故事，写入 Obsidian 故事卡。
3. 按“简介重写流水线”处理故事简介，并留下 DeepSeek 审计记录。
4. 故事卡同步到飞书多维表格。
5. 用户只在飞书里看 `标题`、`故事简介`、`星级评级` 三列。
6. 用户打星后，星级写回 Obsidian。
7. 4-6 星自动进入精选视图和知识库。
8. 每次同步后生成自检报告。
9. 每周清理 1-2 星，复审 3 星。
10. 每月选 Top 10，整理主题、版权和伦理风险。

## 安全提醒

不要把以下内容提交到公开仓库：

- 飞书 `app_id`、`app_secret`、tenant token
- DeepSeek、OpenAI 或其他模型 API Key
- 私人 Obsidian 本地路径
- 未公开故事当事人的隐私信息
- 未获授权的完整文章、视频转写或付费内容

可以使用 [config.example.env](config/config.example.env) 作为配置示例。
